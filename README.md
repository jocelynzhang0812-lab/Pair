# Pair

Agent-to-Agent 人脉管理挖掘平台

> 替你 24/7 筛人、破冰、初步对齐。Coffee chat、找合伙人、找 mentor，不再凭一面之缘。

### 12 个月目标

验证 A2A 对话是否能显著提升职场人脉匹配中“见对人”的效率，并跑通一个可自然增长的最小双边网络。

## Why Pair?

职场人脉匹配的摩擦，不是信息太少，而是匹配太重。

无论是 coffee chat、找合伙人、找 mentor、跨界打听某行业，还是寻找“做过 X 事的人”问 30 分钟，本质都是同一个动作：

> 和一个你不熟、但可能对你很重要的人，建立第一次有效对话。

我们访谈了 10+ 潜在用户，包括 AI 创业者、产品 / 设计 / 技术从业者、实习生。最让我们意外的共识是：找人聊天的“前置过程”心理负担极大。

领英、脉脉、小红书已经提供了足够多的资料和人选，但大多数人的真实场景是：

- 加了好友不知道怎么开口。
- 怕被觉得“功利”。
- 怕浪费对方时间，反而坏了印象。
- 好不容易发了消息，对方不回。
- 聊完才发现不对，过程重、慢、依赖社交勇气。

ASQ 相关研究表明，networking 会让人产生不适和“脏”的心理联想。

> 公司的大佬我不知道他们感兴趣什么，想约咖啡但是害怕被拒绝。
>
> 小龙，Kimi PPT 初级产品经理

> 人脉当然要维护，但 every day 关注别人在干什么，对什么感兴趣，然后选择要不要给他发问候的消息太耗费时间了，ROI 太低。
>
> Bruce Huang，AI 社交产品连续创业者

很多高价值的弱关系链接，在“开口前”就死了。

职场人最缺的不是认识更多人，而是一个低尴尬成本的破冰中介。

## 产品切入点

Pair 从 coffee chat 起步，因为它动机最纯粹、阻力最小、可重复频次最高，最适合验证 A2A 范式。

跑通后，Pair 可以自然延展到更多职场连接场景：

- 找联创。
- Mentor 配对。
- 行业 peer 互助。
- 找“做过 X 的人”问问题。

这些场景共用同一套底层能力：让 Agent 先替人谈一轮。

## Pair 怎么工作

Pair 的切入点不是再做一个资料库，而是让 Agent 先替人完成匹配前的意图澄清。

核心流程：

1. 用户授权后，AI Agent 读取公开资料并生成个人档案。
2. 系统根据用户目标自动匹配潜在人选。
3. 双方 Agent 进行 5 轮结构化对话，共 10 条消息。
4. 对话结束后生成 Summary。
5. 用户根据 Summary 判断“这个人值不值得见”。

Pair 不替代真人社交，而是把匹配和意图澄清前置，让真人见面发生在更高质量的起点上。

## 核心亮点

### 人脉画像

Pair 会“阅读”联系人过去的社交发言、互动记录等内容，自动生成兴趣标签、职业背景、互动节奏等维度信息。

### 会前准备

见面前，Pair 会自动生成简明 briefing，包括共同经历、近期动态、推荐话题，让用户聊天不尬场。

### AI 互动提醒

Pair 会定期提醒用户哪些人很久没联系了、谁今天生日、谁可能需要回应，从此告别“想联系却不知道怎么开口”的尴尬。

## Why Now?

### 技术拐点

LLM 已经能稳定支撑 Agent 间的结构化对话。以 Deepseek V4 为例，一次 10 轮 A2A 对话的模型调用成本已降到可忽略水平。

Agent 框架，如 LangGraph、AutoGen 等，也成熟到可商用。A2A 第一次有了工程基础。

### 范式空窗

中国市场目前没有任何职场社交产品是为 Agent-to-Agent 原生设计的。

主流职业社交产品仍以人和人的资料展示、推荐为核心。A2A 原生的职场连接产品尚未出现明确赢家。

### 用户心智成熟

经过 ChatGPT / Claude 一年多的普及，从 Copilot 到各类 AI 代理工具，用户已经习惯把部分决策前置交给 AI。

职场人对“AI 提效 / AI 代表我做事”的接受度，也远高于普通人群。

## 竞品与机会

最直接的对手是 Lunchclub。

Lunchclub 的 1 亿美元估值验证了赛道存在，也验证了用户接受 AI 撮合。但它有三个限制：

1. AI 只是隐形匹配引擎，没有被塑造成产品角色，缺少差异叙事和情感粘性。
2. 烧了 2800 万美元以上，商业化始终没有跑通。
3. 双边都被动，没有“Agent 替你主动行动”的增量。

这三点正是 Pair 的窗口。
