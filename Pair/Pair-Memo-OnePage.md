## Part 1 · 基本信息

**项目名称**：**Pair**（小红书孵化 · Agent-to-Agent 职场连接平台）

**一句话描述**：每个职场人都有一个 AI Agent，替你 24 小时筛人、破冰、初步对齐——从 coffee chat 起步，把小红书"看到 ta 想认识"的隐性场景做成产品。

**12 个月目标**（去 C 端付费，全部为战略指标）：
- 5 万小红书内注册用户、1 万周活 coffee chat 用户、累计 10 万次真 coffee chat 完成
- M+3 留存 ≥ 35%；A2A → 真见转化率 ≥ 35%
- 至少 3 个垂类社群跑通（AI PM / 设计师 / 创业者 / 港大校友等）
- 战略贡献：为小红书新增"职场关系层"用户分层；高净值职场创作者活跃度可量化提升

**所需资源**：
- **小红书内部**：内容标签流量倾斜入口、用户授权数据接口、AI infra / 模型 API、合规框架支持、产品 / 设计 1 名 + 后端 1 名 HC
- **外部**：技术合伙人候选触达通道、垂类 KOL 共建合作

## Part 2 · Memo（约 2,000 字）

### 为什么要做

职场上和一个对的人聊一次，是被反复印证、长期没被真正解决的事。无论是 coffee chat、找合伙人、找 mentor、跨界打听某行业，还是寻找"做过 X 事的人"问 30 分钟，本质都是同一个动作——**和一个你不熟、但可能对你很重要的人，建立第一次有效对话**。

我们访谈了 30+ 职场人（创业者、大厂中层、产品/设计/技术从业者），最让我们意外的共识是：**找人聊天的"前置过程"心理负担极大**。不是找不到人——脉脉能搜、朋友圈能转介、小红书能私信——而是加了好友不知道怎么开口、怕被觉得"功利"、怕浪费对方时间反而坏了印象、一次没聊好就再也不敢约第二次。

结果是：高价值的弱关系链接，在"开口前"就死了。**职场人最缺的不是认识更多人，而是一个低尴尬成本的破冰中介。**

### 为什么是 Agent-to-Agent，且 Agent 必须是"同事"而非"chatbot"

这件事必须由 AI Agent 代理完成。人做不了（心理成本高、社交人设包袱重）；算法推荐做不了（"猜你认识"解决不了"为什么聊"）；HR / 猎头做不了（成本高、覆盖窄）。

只有 Agent 能做到三件事：替你完整表达背景与诉求 / 24 小时主动连接 / **双方 Agent 先对齐再决定是否让本人出场**。人不必出现在第一轮接触里——这是对"社交尴尬"的根本解构。

**更关键的判断是：Agent 的产品形态必须是"同事"，不是"chatbot"。** 市面上几乎所有 AI 产品默认是输入框 + 对话流——用户问，AI 答。这让用户重新承担思考 + 措辞 + 打字的劳动，本质还是工具。Pair 必须反过来：**Agent 主动扫、主动撮、主动起草，把成果放到桌面上等你审；用户做的只是按按钮接受、拒绝或调整**。零打字。这才是"AI 是同事"的真正落地。

### 为什么是现在

技术拐点（LLM 可稳定支撑 Agent 间结构化对话、调用成本可承受、Agent 框架商用成熟）、范式空窗（没有任何主流产品为 A2A 原生设计）、用户心智成熟（ChatGPT/Claude 普及后，职场人对"AI 代表我做事"的接受度已超阈值）——三件事在 2025–2026 同时发生。这件事去年做要 18 个月教育市场，今年做 6 个月就能跑通。

### 为什么在小红书孵化（核心章节）

Pair 可以在任何地方做，但**在小红书孵化能比独立创业快 6–12 个月跑到 PMF**。理由有五：

1. **流量入口完全免疫冷启动困境**：小红书内 AI / PM / 设计 / 创业 / 留学等垂类标签下有 2,000 万+ 活跃职场内容创作者和读者。Pair 不需要买量，靠内容标签精准引流即可在 3 个月内做到 5 万真实注册——独立产品冷启动需要至少 6 个月和数百万元投放才能拿到同等量。

2. **画像数据的根本性 unfair advantage**：小红书用户的发文是 first-party 数据（背景、能力、关注、表达风格、专业判断），比任何 LinkedIn / 脉脉的抓取数据准至少 10 倍。Agent 建档的质量是 A2A 撮合的基础——这一项小红书有，全市场没人有。

3. **场景天然闭环**：小红书天然存在"看完一篇职场内容想认识作者"的隐性需求（评论区互留微信、私信请教是高频但低效），但破冰心理负担没人解决。**Pair = 把这个高频隐性场景做成显性产品**。这条 funnel "看内容 → 想认识 → Agent 替你破冰 → 真聊"是独立产品永远拿不到的。

4. **品牌信任降低 Agent 代社交的接受门槛**：让用户把"AI 代我社交"的权限交给一个陌生独立 App，需要 6–12 个月信任建立。挂在小红书 backed 的子产品下面，第一天就有信任基础。

5. **基础设施复用省 6 个月工程**：AI infra、模型调用、合规框架、客服体系、支付（未来收费时）、风控全部可复用。Pair 团队不必重新搭这些。

而 Pair 给小红书的战略回报同样清晰：(a) 把"内容消费"升级为"内容消费 + 关系行动"，提升高净值职场用户的停留时长与留存；(b) 切入"职场关系层"新场景，对标 LinkedIn 的市场空间但路径完全不同；(c) 为小红书 AI 战略提供 A2A 范式的实验场，沉淀的对话数据与 multi-agent 编排能力可反哺小红书核心 AI 助手。

### 与小红书既有产品的关系（防竞合）

Pair 不动 feed、不动现有私信 / 群聊。它是独立子产品形态：**独立小程序 + 小红书 App 内的 H5 入口卡**。数据是单向授权关系——Pair 读用户的小红书画像（一律 opt-in），Pair 自己产生的撮合 / 对话数据**不污染 feed 推荐**。中长期可叠加为 feed 的"行动层"：浏览某内容时显示"和 ta 的可能撮合度"，但这是 Phase 3 才考虑的延展，不是 Phase 1 必须做的事。

### 场景：以 coffee chat 起步，但终点是职场连接全景

Pair 的核心机制可以服务很多"想见一个对的人聊一次"的场景，但我们刻意**只从 coffee chat 起步**——动机最纯粹、阻力最小、可重复频次最高，最适合验证 A2A 范式。跑通后自然延展：找联创 / Mentor 配对 / 行业 peer 互助 / 找"做过 X 的人"问问题 / 私董会撮合。所有场景共用一套底层。

**辅助场景（极简）**：用户通过 Pair 见过的人会自动沉淀进"我的 Connections"，Agent 监测对方升职 / 融资 / 公开发文等关键事件并触发 reach-out 卡片。保证用户用 Pair 建立的关系不会死掉——这是与 Lunchclub 这类"一次性撮合工具"的根本区别。

### 竞品坐标

按"目标 vs 陪伴"和"AI 是手段 vs 终点"两轴切分四象限：Tara/Replika 是娱乐陪伴 × AI 为终点；领英/脉脉是职业 × 展示型；SecondMe 是 AI 身份协议层；Lunchclub 是 AI 推荐但范式仍是 chatbot/被动撮合。**Pair 独占"职业目标 × Agent 是手段"右下象限**。

最直接对手 Lunchclub（a16z + Coatue 投，估值 $1 亿+）验证了赛道，但有三个致命限制：AI 只是隐形匹配引擎缺差异叙事 / 烧 $28M+ 商业化没跑通 / 双边都被动缺主动行动。**这三条正是 Pair 的窗口。**

会被问到的两个 killer question 提前回答：
- **为什么不卖成小红书的一个功能**？因为 A2A 是新交互范式，受 feed-first 架构约束的功能补丁会变成"附属功能"，无法承载"AI 替我社交"所需的新心智。子产品形态可以独立沉淀产品性格、撮合算法、A2A 对话能力，未来再决定是否回灌主 App。
- **LinkedIn / 脉脉为什么不会复制**？他们的"人-人"展示型架构不支持 A2A，任何补丁式 AI 都会让用户感觉"假装是我的 AI 在回复"——这反而破坏既有关系。新范式需要新平台，且需要双方都默认对方是 Agent 的纯净环境。

### 为什么是我

- **深度场景理解**：港大 CS 在读，已有两段顶尖 AI 公司 PM 实习（Moonshot Kimi Agent 方向、万兴科技 AIGC 视频），亲历 coffee chat / 找 mentor / 跨圈引荐的全部痛点
- **产品 + 技术复合**：能独立完成产品定义、Figma、用户体验全流程；CS 背景给 Agent 架构 first-principle 判断
- **小红书原生用户 + KOL 资源**：运营 1.3w 粉小红书账号，对小红书内容生态、职场创作者圈层有 first-hand 理解，自带冷启动传播能力
- **诚实承认短板**：不是后端 / 算法重度背景，技术合伙人是关键变量

### 团队与时间承诺

- **章璟菲（创始人）**：明确全职投入 Pair 项目，学业排期已与导师对齐；M0–M3 投入 ≥ 80% 时间
- **技术合伙人**：3 位候选评估中，目标 M2 前确认人选并启动
- **首批 HC（M0–M3 需要小红书配置）**：产品 / 设计 1 名 + 后端 1 名，可来自小红书内部转岗或外招
- **M6 团队画像**：6–8 人核心团队（产品 2、设计 1、后端 2、增长 1、运营 1–2）

### 已经做了什么

不是 PPT 项目，而是**已有可跑通的 working product**：

1. 30+ 用户访谈，定位"破冰前置心理负担"核心痛点
2. **CoffeeChat Agent v0.5 已上线运行**（Vercel 部署，React + tRPC + Hono + Drizzle + MySQL + DeepSeek）：单用户开箱即用模式跑通
3. **非 chatbot 形态已验证**：Today 流 + 卡片化推送 + 命令面板模型已实装，"Agent 当同事"范式可工程化
4. 数据基建可直接平移为 Pair 双边架构
5. **小红书 scanner 已跑通**：DDG snippet 抓取链路 + 事件识别可直接复用——这是 Pair 与小红书母体衔接的关键技术基础
6. 种子档案策略已成型，可作为 day 1 内测数据

**关键依赖与风险**：CTO 招募节奏（影响 Phase 1 起跑）/ 小红书 OAuth 与画像 API 排期 / A2A 合规二次审查 / 与 feed、AI 团队的跨组协作（影响 Phase 3 反哺路径）。

商业化（B2B 白标）作为 Phase 3 P1 探索性 POC，**不进 12 个月闸门**——OPC 项目的第一优先级是给小红书创造可量化的战略价值，不是自己赚钱。

### 最大的风险与应对

**风险 1 · Agent 对话质量不可控**——Agent 代社交，万一对话出格、误传诉求、留下糟糕印象，破坏的是用户真实职业关系。应对四层：三段式护栏（Agent 对齐 → 用户预览 / 批准 → 人接管）/ 可观测可干预 + 敏感内容自动 redact + 1% 抽审 / 场景收窄到 coffee chat 单一动机 / 小步放权（写开场白 → 半自动 → 全代理）。

**风险 2 · 合规与数据授权**——A2A 代理涉及用户授权、PII redact、未成年保护、训练数据授权，国内合规深度足够（网信办算法推荐管理规定 + 即将出的 AI 监管法规）。应对：复用小红书既有合规框架；用户授权一律 opt-in；与小红书法务在 M1 完成 Pair 专属合规审查；数据训练用授权可关闭。

### 7 月 15 日后的 Demo 期路径

1. **Demo 打磨**：基于现有 CoffeeChat Agent + 小红书登录与画像接入，跑通 coffee chat 单场景闭环（主屏 Match Card 流，非对话流）
2. **真实用户测试**：邀请 30–50 位小红书 AI PM / 设计师 / 创业者圈种子用户跑两轮真撮合，拿首批 A2A 反馈
3. **计划书补全**：HC 预算、单位成本（API 调用、撮合成本）、与小红书核心指标的协同模型
4. **关键招募**：技术合伙人 + 内部 HC 触达
